About

SheetMusic++ is a web application developed in 2021 by Kian Chamine, James Karsten, Hilary Nelson, and Jack Weiler that allows users to upload an image of sheet music for digitization into three separate formats. For detailed information on how to run the app, its limitations, and credits, please see the README on GitHub. The three output file types are:

  1. PDF: This is a highly compatible document that can be read on most machines and physically printed. It has a small file size that allows for easy storage and sharing.
  2. LilyPond: This is a digital music composition markdown format (similar to LaTeX) that can be edited using any plain text editor. To compile it into sheet music, either download LilyPond and install it on your personal machine or copy and paste the code into an online editor such as LilyBin or HackLily.
  3. MIDI: This stands for Musical Instrument Digital Interface, which is a widely-used, editable music format that stores complex notation in a small file size. Both the notes and overall information such as tempo can be adjusted, and the track can be rendered using thousands of synthesized and sampled instruments. Programs such as GarageBand and Logic Pro can edit and export MIDI tracks into various audio formats such as MP3 or WAV. There are also free online compilers such as signal and SolMiRe that can edit basic features of MIDI tracks. Read more here.

Our web app utilizes two models: an open-source semantic model designed to read an individual line of music and translate that to a semantic vocabulary, and a YOLO model that we use to separate multi-lined scores for the semantic model to read one by one. These separated lines are then parsed back together into our final PDF, MIDI, and LilyPond formats.
